# Claude Code 実装指示文：財務因果DAG構築・調整システム（プロトタイプ）

## あなたの役割
これから「財務業績分析のための因果DAG自動構築・調整システム」のプロトタイプを実装する。以下は仕様の要約と、実装を進める際の優先順位・制約である。フルスペックを一度に作り切ろうとせず、**フェーズごとに動くものを作って都度確認を取りながら**進めること。

## プロジェクトの目的（要約）
- 財務三表（PL/BS/CS）＋非財務KPIの因果関係をDAGとして表現する
- 「業界標準テンプレート（LLM生成）」×「企業固有IRデータ」×「ユーザーの分析ゴール」をマージしてDAGを作る
- DAGをインタラクティブなツリーUIで可視化する（プラス影響=緑実線、マイナス影響=赤実線、AI仮説=点線）
- ユーザーとAIが5ラウンドの対話（説明＋質問形式）でDAGを調整・確定し、以降は任意追加ラウンドで再調整できる

## 推奨技術スタック
- LLMオーケストレーション: LangChain または LlamaIndex
- フロントエンド可視化: React + React Flow（またはD3.js）
- 因果推論バックエンド: Python + DoWhy
- LLM API: Anthropic Claude API（モデルIDは `claude-api` スキルで最新のものを確認して使うこと）

## 実装フェーズ（この順番で進め、各フェーズ完了ごとに動作確認してから次に進むこと）

### Phase 0: プロジェクト初期化
- モノレポ構成を提案（例: `backend/`（Python, DAG・LLM・DoWhy処理）、`frontend/`（React + React Flow）)
- DAGのデータモデルをまず設計する：ノード（財務科目/KPI）、エッジ（符号・強度・確信度・ラグ・出典＝AI仮説かユーザー確定か）をJSON Schemaとして定義する
- ここで一度、ディレクトリ構成とデータモデルをレビューしてもらってから次に進むこと

### Phase 1: 業界標準DAGテンプレート生成（①）
- ダミーの業界（例：製造業1業種のみ）を対象に、LLMにPL/BS/CS間の標準的な因果関係を生成させ、Phase0のデータモデルに従ったJSONとして出力する機能を実装
- まずは1業種だけで通し、他業種への拡張は後回しにする

### Phase 2: 因果ツリー可視化UI（④）
- Phase1で生成したJSONをReact Flowで表示するだけの最小UIを作る（緑実線／赤実線／点線の描き分けを実装）
- 実データ収集より先にUIの見た目とインタラクション（ノードクリックで詳細表示、等）を確定させる

### Phase 3: 企業IRデータ取り込み（②）
- **注意：実際のEDINET/企業IRサイトへの自動巡回スクレイピングは、対象サイトの利用規約・robots.txt・アクセス頻度の妥当性を確認できるまで実装しない。** まずは「手元に置いたサンプルPDF/HTMLファイルを解析してKPIを抽出する」ローカルファイル入力版から作ること
- 手動データ登録（CSV/Excelアップロード）UIも合わせて実装
- 実サイトへの自動収集を本当に実装する段になったら、対象サイト一覧と頻度制限を先にユーザーに確認すること

### Phase 4: マージ・カスタマイズ機能（③）
- 「分析ゴール」入力欄と、非財務データをドラッグ＆ドロップでツリーにマッピングするUIを実装
- Phase1のテンプレート＋Phase3の企業データをマージしてDAGを更新するロジックを実装

### Phase 5: 5ラウンド対話チューニング（⑤）
- AIが「説明文＋Yes/No等の質問」を1問ずつ提示し、ユーザー回答に応じてDAGのエッジ（符号・強度・ラグ）を更新するチャットUIを実装
- 5ラウンド終了後は「ロック」状態にし、以降は自由入力プロンプトで追加調整ラウンドを開始できるようにする

### Phase 6: DoWhy連携（因果効果の検証）
- 確定したDAG構造をDoWhyに引き渡し、サンプル数値データで因果効果を計算する最小限の検証パイプラインを実装

## 進め方に関する制約
- 各Phaseの実装後、簡単な動作確認（UIならブラウザで実際に操作、バックエンドならサンプル入力での実行）を行ってから次のPhaseに進む
- 外部ネットワークアクセスを伴う機能（実サイトのスクレイピング、外部LLM APIへの本番キーでの呼び出し等）を追加・実行する前には一度立ち止まってユーザーに確認する
- 仕様にない機能や過剰な抽象化は追加しない。まずは各Phaseの最小動作版を優先する

## Phase 0 で最初にやってほしいこと
1. モノレポのディレクトリ構成案を提示する
2. DAGノード/エッジのJSON Schemaを設計して提示する
3. 上記2点についてこちらの合意を取ってから、Phase 1（1業種分の標準テンプレート生成）の実装に着手する
